# backend

### Branch
- Branch 관리
  - Main Branch
  - Dev Branch: For Development

- Branch 생성 규칙
  - Default: `작업태그/Name`
  - ex) `feat/HomeView`

### Pull Request 
- PR(Pull Request) 규칙
	- main 브랜치에 병합 시 필요
  - 작업 사항, 스크린샷, To Reviewers 등 활용
	- Default: `[날짜] 이유영 - 작업 사항`
	- ex) `[11/8] 이유영 - 회원가입 뷰 추가`

- 작업태그 일괄
	- `Feat` : 새로운 기능 추가 / 일부 코드 추가 / 일부 코드 수정
	- `Fix` : 버그 수정
	- `Refactor` : 코드 리팩토링
	- `Design` : 디자인 수정
